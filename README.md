# Database Design & Entity-Relationship Modeling

**Author:** Zanyar Erkozan

---

## 📋 Overview
This project focuses on the conceptual design phase of database systems. It involves translating real-world requirements into robust **Entity-Relationship (ER) Diagrams**.

## ⚙️ Core Concepts Covered
- **Entity Identification:** Defining strong and weak entities.
- **Relationship Mapping:** 1:1, 1:N, and M:N relationship cardinality and participation constraints.
- **Attributes:** Handling composite, multivalued, and derived attributes.
- **Database Schema Translation:** Converting high-level ER diagrams into normalized relational database schemas.

## 📁 Files Included
- `Database-Design-Assignment-1.pdf`: Detailed documentation, problem statements, and completed ER diagrams.
